# About_LeastSquaresMethodInRate
	목적 함수(Objective Function)의 정의: 잔차 제곱합(RSS) 코드는 4개의 실수 데이터 포인트(만기 x, 이자율 y)를 가장 잘 설명하는 모델 f(x)를 찾는 과정입니다. function f(x;β)=(a+bx)e^-cx+d
(여기서 β=[a,b,c,d]는 찾아야 할 모수 벡터.) 최소제곱법의 목표는 실제 관측치 yi와 모델 추정치 f(xi;β) 간의 오차(잔차)를 제곱하여 모두 더한 값, 즉 잔차 제곱합(Residual Sum of Squares, RSS) S(β)를 최소화하는 모수 벡터 β를 찾는 것입니다. S(β)=∑for i=1 to n (yi−f(xi;β))^2
	선형 회귀와의 차이: 왜 비선형 최적화가 필요한가? 만약 모델이 선형이었다면, 선형대수학의 정규방정식(Normal Equation)인 β=(X'X)^−1X'y 행렬 연산 한 번으로 정확한 해를 단번에 구할 수 있다. 하지만 모델에는 e^−cx라는 비선형 항이 존재합니다. 미지수 c가 지수(Exponent) 자리에 있기 때문에, 행렬의 역행렬을 구하는 해석적(Analytical) 방법으로는 해를 찾을 수 없다. 따라서 초기 추정치에서 출발해 점진적으로 오차를 줄여나가는 수치해석적 반복 최적화(Iterative Numerical Optimization)가 필수적입니다. 이것이 curve_fit이 수행하는 진짜 역할이다.
	핵심 수학: 야코비안(Jacobian) 행렬과 경사하강 curve_fit은 내부적으로 레벤버그-마쿼트(Levenberg-Marquardt) 알고리즘이라는 고급 수학 기법을 사용합니다. 이는 오차 함수 S(β)의 극솟값을 찾기 위해 각 모수(a,b,c,d)에 대한 편미분을 수행하여 다변수 함수의 기울기, 즉 야코비안(Jacobian) 행렬을 계산합니다.
모델을 각 모수로 편미분하면 다음과 같은 1차 도함수 벡터가 도출됩니다. 
∂f/∂a=e^−cx
∂f/∂b=xe^−cx
∂f/∂c=−x(a+bx)e^−cx
∂f/∂d=1 

알고리즘은 이 편미분 값(기울기)을 바탕으로, 테일러 전개(Taylor Expansion)를 통해 비선형 함수를 국소적으로 선형화한 뒤, 기울기가 0이 되는 방향(KKT 조건의 1차 필요조건)으로 파라미터 벡터 β를 미세하게 업데이트합니다. 이 과정을 잔차의 변화량이 거의 0에 수렴할 때까지 반복하여 최종 최적 모수 a,b,c,d를 산출하는 것입니다.
